Inheritance

What is Inheritance

It's all about relationships

Inheritacne is a very importance concept in object oriented programming. Java allows classes to inherit other classes and use things such as methods and varibles from the other classes. The benefits of using inheritance is to create new classes that is build upon other classes.

Each class in Java is allow to have one superclass, "a parent" and each class could have unlimited subclasses, "children".

Inheritacne creates a Is a relationship. between classes, the "Is a relationship" is also called the Parent - child relationship.

The extend keyword is used to create interitance. For example if class chocolateIcecream extends Incecream,
then icecream is the parent of chocolate icecream and chocolate icecream is the child of icecream.

Abstraction

What is Abstraction

Hiding the details

Start off, we should be clear that abstraction is not actually the same as abstrac class. Abstraction is just an idea of hiding the implementation from the user, and only the functionality is presented
For example every time you start a computer do you think of how the computer actually starts from the momment current is being pass to the electronics of the computer?
I think the answer for most people is no, we know if we push start button the computer will turn on.
That is what an idea of abstraction is. The computer is built base on layers of abstractions from both hardware to software.

In Java people use Interface and abstract classes to implement the idea of abstraction. As we already know from the other page interface's functionality and but we haven't talk about abstract class yet and that is what we will discuss today.

An abstract class is defined with the keyword abstract. For example Public abstract class icecream . An abstract classs is not an like the previous classes that we have created. It could not be instantiated. Instead if you want to use it you need to make it complete or concrete by extending it with a subclass.
We call a class concrete class if it doesn't have any abstract methods and it is implementing the abstract class or interface.

An abstract method is just a method with no functionality defined, but only the return type and the input arguments. If you want to use abstract methods they have to be in an abstract class. More details will be shown in the code examples

Debugging

When programming, the programmer will often find out that their program doesn't work as they have expected.This is because ideas and code don't translate 100 percent of the time.

A good way to prevent much of the error is to write out a complete pseudocode of you program before the actual coding. Also when programming a big program it would be nice if you test your segments of your code when you have chance because it is easy to find bug in a 10 line code than a 1000 line of code.

Below is the steps you can take before debugging

  • Step 1 : Check if the bug is real, i.e run the program a few more time to check if the bug is realy being produced
  • Step 2 : Check your error log. Most IDE will have a "stack traces" that tells your what the bug is, and what line it is found. Use that to your advantage
  • Step 3 : Use debugger and print statement to see what is going on in your code.
  • Step 4 : If you cant find bugs from just step 3, try writing a test case for your error. The test case should be designed to check the result of a segment of code, and should be short and clear.
  • Step 5 : Literlly google what might cause your specific error.
  • Step 6 : If all the above fail, see help from your LA or professor.
